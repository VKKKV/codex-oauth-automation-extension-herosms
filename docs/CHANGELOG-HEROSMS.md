# HeroSMS Fork 更新日志 (2026-04-22)

本分支（HeroSMS Fork）在 Pro5.8 基础上进行了深度定制，旨在解决自动化注册中的手机号验证瓶颈与 163 邮箱兼容性问题。

## 主要变更

### 1. HeroSMS 手机号验证集成
- **全自动流程**：当检测到 `add-phone` 页面时，系统会自动启动手机验证逻辑。
- **动态调价**：支持自动获取接口最低价，并在价格波动（WRONG_MAX_PRICE）时自动按建议价重试，极大提高了买号成功率。
- **号码复用**：支持在 activation 有效期内复用同一个手机号，节省接码成本。
- **自动填写**：自动剥离国家代码前缀，并将号码与验证码精准填入 OpenAI 认证页。

### 2. 163 邮箱深度适配 (协调器模式)
- **跨 Frame 提取**：针对 163 邮箱将正文加载在 cross-origin iframe 的特性，实现了“协调器”架构，确保脚本能看穿 iframe 提取验证码。
- **智能去重**：实现了基于内存与 Session 存储的双重去重机制，绝不误抓旧验证码。
- **强制延迟同步**：为 Step 8 (登录) 增加了 10 秒强制等待与列表刷新逻辑，确保 NetEase 后端推送完成后再开始读取。
- **标题秒提**：支持从邮件标题（Subject）直接提取验证码，无需点击邮件即可秒速完成验证。

### 3. 自动化链路增强
- **智能快照绕过**：如果邮件时间戳明确晚于请求时间，系统将绕过“旧邮件快照”，实现秒速响应。
- **SPA 状态稳定性**：优化了 163 邮箱 SPA 切换时的等待逻辑，增加 2-2.5 秒的“列表沉降时间”，彻底解决“元素未定义”报错。

### 4. 项目清理
- **移除商业信息**：删除了原项目中的打赏码、QQ 交流群及相关商业宣传文档。
- **许可证变更**：将项目许可证由 Apache 2.0 变更为 **GPLv3**。
